## 0. 一句话电梯陈述

> **S3FM** 把 S3GM 的"预训练时空先验 + 推理时测量引导"框架保留下来,但用 **flow matching** 替换 1000 步 score-SDE 采样;真正的杠杆不在"换采样器",而在于让流的**源分布带上目标信息**(一张廉价的粗化场),使得传输路径更短,从而 ≤50 次模型评估就能逼近 S3GM@1000 步的重建质量。

## 1. 研究问题

给定一个时空动力系统(首个验证对象:Kuramoto–Sivashinsky 方程,KSE),我们只能观测到它的**稀疏 / 含噪 / 不完整 / 非线性**测量:

```
y = H(X) + 噪声
```

- `X`:未知的完整时空场(全部帧、全部物理量)。
- `H`:已知、可微的测量算子(稀疏点采样、规则下采样、傅里叶子采样、时间平均、速度幅值……)。
- 目标:恢复完整的 `X`,并能向观测区间之外**预测**。

**核心研究问题:**

> 一个从"有信息量的源分布"出发传输的预训练 flow-matching 先验,能否在**不为每个 `H` 重新训练**的前提下,**显著快于 S3GM** 地重建并预测完整时空动力学;**而且这份加速可被证明来自更短的传输路径,而不只是把 SDE 换成 ODE?**

最后半句是这个项目的灵魂——它把"看起来像加速"和"真的有方法贡献"区分开。

## 2. 出发点:两篇论文

这个 idea 是把两篇论文嫁接起来的产物。

### 2.1 S3GM —— *Learning spatiotemporal dynamics with a pretrained generative model*

提供了整套"用生成先验做稀疏重建"的范式,可复用的零件:

- 无条件**时空生成先验**(把"什么是合理的动力学"学进网络);
- 推理时的**观测一致性**引导(`y` 和 `H` 在采样中起作用,先验不动);
- **并行重叠窗口**——把长序列拆成有重叠的窗口、靠重叠区一致性拼接;
- **自回归续推**——超出观测区间后,用前一段的尾帧作条件继续往后生成;
- 多次采样得到**不确定性**。

**它的痛点(我们要打的靶):** 重建靠一个 **1000 步的引导 reverse-SDE**,高维场上太贵。

### 2.2 *On the Guidance of Flow Matching*

提供了"如何在 flow matching 里做测量引导"的理论与方法:

- 一个对**任意源分布、甚至依赖耦合**都成立的通用引导向量场;
- 干净端点估计 `X1_hat`;
- 协方差梯度引导 **`g_cov-G`**(主方法)和它的快速近似 **`g_cov-A`**;
- 一个关键的**反向耦合比** `P = π'(x0|x1)/π(x0|x1)`:独立耦合下 `P=1` 精确成立,对依赖耦合也是合理近似。

**对本项目的关键意义:** 正是 `P` 这一项,在理论上**许可了我们使用非高斯、依赖耦合的源**,同时复用同一套闭式引导公式。我们刻意工作在 `P ≈ 1` 范围内,不去碰 `P ≠ 1` 的新理论。

## 3. 核心 idea:S3FM = S3GM 的框架 + Flow Matching 的引擎

| | S3GM | **S3FM** |
|---|---|---|
| 先验 | score(分数)先验 | **velocity(速度)先验** |
| 采样 | 引导 reverse-SDE | **引导 forward-ODE** |
| 步数 | ~1000 步 | **目标 ≤50 NFE** |
| 引导 | DPS 式分数引导 | **`g_cov-G` 协方差梯度引导** |
| 算子无关 | 是 | **保持是** |

但**仅仅这样替换不构成方法贡献**:在"标准高斯源 + 线性路径 + 独立耦合"下,flow matching 在数学上等价于一个仅差 noise schedule 的 diffusion。所以——

#### 为什么"高斯版 = diffusion 换皮"?(把这句话讲透)

Diffusion 的前向过程,就是给干净数据**逐渐掺高斯噪声**;而高斯源线性 flow 的中间状态,展开后是同一个形状:

```
Diffusion 中间态 :  x_t = α_t · X + σ_t · ε        (ε 是高斯噪声,α/σ 是 noise schedule)
高斯线性 FM 中间态:  Zs  =  s  · X + (1−s)· ε        (因为 Z0=ε, Z1=X, Zs=(1−s)Z0+sZ1)
```

两式**形式完全一致**——都是"缩放的干净信号 + 缩放的高斯噪声"。线性 FM 不过是选了一个特定 schedule(`α=s, σ=1−s`)。于是:

- 两者每一时刻的中间分布属于**同一族**("数据 + 某个信噪比的高斯噪声");
- 彼此只差一个对时间轴的**单调换元**(这个换元就叫 noise schedule);
- 速度 `v` 与 score `∇log p` 之间有确定的代数互换公式;
- diffusion 的 **probability-flow ODE 本质上就是一个高斯源的 flow**。

**结论:** "把 score-SDE 换成 velocity-ODE" 不是新模型,是**同一个东西换身衣服**;唯一真正的差别只是采样器(ODE vs SDE),而这个差别 diffusion 自己内部(probability-flow ODE)早就有了。**这正是为什么高斯版只能当对照基线(S3FM-Gauss),也正是为什么必须加 `S3GM-PF-ODE` 归因基线把"ODE vs SDE"这点已知差别单独减掉(见 §5)。**

#### Flow matching 比 diffusion 多出来的能力,恰恰是 S3FM-Info 踩的点

| | Diffusion | Flow Matching |
|---|---|---|
| 源分布 | **只能是高斯**(前向被钉死成"加高斯噪声") | **可任意非高斯**(例如 `S(X)`) |
| 耦合 | 数据配**独立**噪声 | 可配**依赖**源(每张模糊配它自己的清晰) |

Diffusion 根本没法从一张结构化的粗化场出发——而这正是下面 S3FM-Info 的全部要点。**高斯版=走在 diffusion 等价范围内=只能当基线;信息化源版=走出该范围=真正的贡献。**

### 3.1 真正的头号贡献:S3FM-Info(信息化源)

#### 先讲清楚:"源分布带上目标信息"是什么意思

flow matching 生成样本 = **从起点 `Z0` 流到终点 `Z1=X`**。终点是数据,**起点 `Z0` 从哪个分布抽,是你自己定的设计**——这个起点的分布就叫"源分布"。核心问题只有一个:**起点 `Z0` 和它要去的终点 `X`,有没有关系?**

- **高斯源(不带信息):** `Z0 ~ N(0,I)` 是一团纯噪声,里面**没有任何关于 `X` 的线索**。给你看 `Z0`,你猜不出它要变成哪张场。
  > 类比:起点是一张**纯白噪声雪花**,要从雪花里凭空长出完整清晰的图。
- **信息化源(带信息):** `Z0 = S(X)+η` 是 `X` 自己的一张**模糊/降质版**。给你看 `Z0`,你**一眼能看出**它要变成哪张场,只是缺细节。
  > 类比:起点是一张**打了重度马赛克的照片**,轮廓内容都在,只需把细节补清楚。

**"源分布带上目标信息" = 把生成所需的信息,预先塞一大部分进起点里**,而不是让流动过程从纯噪声里全部凭空造出来:

```
高斯源:   [ 起点已有: 0%  ]  +  [ 流动要补: 100% ]  = 完整场
信息源:   [ 起点已有: ~70% ]  +  [ 流动要补: ~30% ]  = 完整场
                 ↑                       ↑
            源“带上”了目标信息       流只需补“缺失的细节”
```

起点已经在终点附近(70% 的活儿起点替你干了),流动只需走完剩下那一小段 → 这就是传输更短、步数更少的来源。

> **"那推理时没有 `X`,怎么算 `S(X)`?"** —— 训练时数据完整,`Z0=S(X)+η` 能直接算,模型学会"从模糊版流到清晰版";推理时没有 `X`,所以 `Z0` 改用测试时拿得到的量构造,但必须在统计上和训练源对齐(详见实现规范的"训练/推理源一致性")。而 `S` 只是廉价的模糊/下采样、不是另一个网络,几乎不花成本——这就是为什么"带信息的起点"代价极小、省下的却是大量 ODE 步数。

#### 形式化

不让流从纯高斯噪声出发,而是从一个**廉价、结构化、带有目标信息的源**出发:

```
S3FM-Gauss(对照基线):  Z0 ~ N(0, I)            纯噪声,不含 X 的信息
S3FM-Info (头号贡献):   Z0 = S(X) + η          S 是廉价的"破坏信息"算子
```

其中 `S` 是一个固定、廉价的腐蚀映射,例如:

1. **粗化-模糊源:** 空间下采样再插值回去(= 对稀疏观测做朴素插值会得到的东西);
2. **谱截断源:** 只保留最低 `k` 个傅里叶模态,其余置零,让流去补细节;
3. **低保真代理源:** 一次快速粗解(留给后续更复杂的系统)。

**为什么这能少步数(将要被实测的机制):**

先理解采样在干什么:生成一个样本 = 从源点 `Z0` 出发,**用数值方法解 ODE** `dZ/ds = v(Z,s)`,`s` 从 0 走到 1,终点就是样本。数值积分就是"走一小步、重看方向、再走一小步",它的误差取决于两件事:

- **要走多远** —— 位移 `‖Z1 − Z0‖` 有多大;
- **路有多弯** —— 速度方向沿途变化多剧烈(轨迹曲率)。

> 路又长又弯 → 必须切成很多小步才不跑偏 → **NFE 多**。
> 路又短又直 → 几步甚至一步就够 → **NFE 少**。

线性路径 `Zs = (1-s)Z0 + sZ1` 下,每个样本要走的位移是:

```
高斯源:   Z1 − Z0 = X − ε          ≈ “整张场”那么大,且路很弯
信息源:   Z1 − Z0 = X − S(X) − η   = 只剩“缺失的细节”(高频结构),位移小得多
```

位移更小只是其一;**依赖耦合**还让路更直——每张模糊场只和它自己那张清晰场配对(而非随机配),于是速度场指向各自明确的目标,而不是"对一堆不同目标取平均"(高斯源早期那个噪声点平均通向无数目标,所以速度场来回改方向 → 高曲率)。

**一个比喻:**

> - **高斯源** = 你被随机空投到地球某个随机角落,要开车到一栋指定的房子:路又长、拐弯又多,得频繁看导航(很多步)。
> - **信息化源** = 你被空投到那栋房子所在的**街区**,离目标只剩几个路口、方向基本是直的:几步就到(很少步)。

位移小 + 路直 → 同样误差下所需的 Euler/RK 步数更少。**这是一个待验证的机制,不是默认假设——M4b 会专门实测平均传输位移 `‖Z1−Z0‖` 和路径曲率,用数据证明它确实更短更直。**

### 3.2 一条不可妥协的红线

源算子 `S` **绝不能依赖具体的推理算子 `H`**。否则"通用先验 + 算子无关引导"的叙事崩塌,模型退化成一个伪装的条件回归器 `y → X`。`S` 定义的是一族通用、固定的腐蚀;推理时的 `(y, H)` 再在先验内部做引导。这条分离是整个工作的卖点,必须在论文里讲明。

## 4. 方法骨架(五块拼图)

### ① 流先验与端点估计

线性路径训练速度网络 `v_theta`:

```
Zs = (1-s)Z0 + sZ1,   目标速度 = Z1 - Z0,   损失 = ‖v_theta(Zs,s) - (Z1-Z0)‖²
```

干净端点估计(Tweedie 的 flow 类比,**与源无关**):

```
X1_hat = Zs + (1-s)·v_theta(Zs, s)
```

S3FM-Gauss 与 S3FM-Info 只在"怎么抽 `Z0`、怎么配对"上不同,路径/损失/端点公式完全一样。

### ② 测量引导 `g_cov-G`

把测量约束写成可微能量 `J`,沿其下降方向修正速度:

```
g_cov-G = -λ(s)·∇_Zs J(X1_hat)
被引导速度 = v_theta(Zs,s) + g_cov-G
```

关键实现细节:梯度**必须同时穿过端点公式和 `v_theta`**(一次穿过网络的反传)——主方法里不许 detach。符号必须用一维玩具测试数值验证("一小步引导应使 `J` 下降"),不能凭论文公式记忆,因为流时间约定不同会翻号。

### ③ 三种能量,分相位使用

| 能量 | 含义 | 用在哪 |
|---|---|---|
| `J_obs` | 观测一致性 `‖y − H(X1_hat)‖²` | 有观测的重建相位 |
| `J_seq` | 重叠窗口一致性(stop-gradient 锚定前窗) | 长序列并行重建 |
| `J_init` | 自回归初始帧一致性 | 观测区间之外的预测 |

**纪律:** 不要把三种能量塞进同一次更新。有观测相位用 `α·J_obs + β·J_seq`;自回归相位只用 `γ·J_init`。

### ④ 并行重叠窗口

长于训练窗口的序列拆成有重叠的窗口,所有窗口作为**同一个联合 ODE 状态**一起积分,靠 `J_seq` 消除接缝。窗口索引用整数帧标签做单元测试,保证既不丢帧也不重复。

### ⑤ 自回归续推

观测用尽后,用上一段的最后 `T_init` 帧作为 `J_init` 条件,一次生成一个新窗口,只追加新帧。混沌系统中逐点会发散,所以**逐点误差和统计误差要分开报告**。

## 5. 怎么证明它"不是换个采样器"(项目的关键设计)

整个项目最难、也最重要的地方是**因果归因**。为此设了两条对照线:

1. **S3FM-Gauss(高斯源对照):** 同样的 flow + 引导,但源是纯高斯。它和 S3FM-Info 的差距,就是"信息化源"单独的贡献。
2. **S3GM-PF-ODE(强制归因基线):** 把**同一个 S3GM score 模型**当作 probability-flow ODE、配相同引导,在 10/20/50/100 NFE 下积分。它把"ODE vs SDE"这一项单独剥离出来。

#### 插话:probability-flow ODE(PF-ODE)是什么?

同一个训练好的 diffusion / score 模型,有**两种采样方式**——PF-ODE 就是其中确定性的那种:

```
reverse-SDE      :  dx = [漂移(含 score)] dt + [扩散] dW    ← 每步注入随机噪声,路径乱抖(S3GM 用这个,1000 步)
probability-flow :  dx = [漂移(含 score)] dt                ← 扔掉随机项 dW,路径光滑确定
ODE
```

**核心事实(Song et al. 2021):** 这两条路形态完全不同(一个乱抖、一个光滑),但它们在**每个时刻 `t` 的概率分布完全相同**——ODE 只是把随机性"积分掉"了。

> 类比:从山顶下山。SDE = 一群醉汉乱走,但各高度的人群密度是固定分布;PF-ODE = 让每人沿"密度场"的光滑流线走,不喝酒——结果各高度的人群密度和醉汉版**一模一样**,但每条轨迹是确定光滑的。

它对本项目的三点意义:
1. **diffusion 自己就能用 ODE 采样**,不需要 flow matching;而且 ODE 路径光滑,通常**比 SDE 步数少得多**。
2. 所以"S3FM 比 S3GM 快"可能**只是因为 ODE vs SDE**,跟 flow matching 无关——这正是必须做 `S3GM-PF-ODE` 对照的理由(把这点已知便宜单独减掉)。
3. **PF-ODE 本质上就是一个高斯源的 flow matching ODE**——这和 §3"高斯版=diffusion 换皮"是同一个数学事实的两面,也是为什么 S3FM-Gauss 只能当基线。

**判定逻辑:**

> 如果在匹配 NFE 下,`S3GM-PF-ODE` 已经能追平 S3FM,那么诚实的结论是:加速来自"确定性积分",而不是我们的方法 —— 这种情况必须明白写出来,而不是埋掉。
>
> 只有当 S3FM-Info **同时**(a) 击败 S3FM-Gauss 且 (b) 击败 S3GM-PF-ODE,并且 (c) 有实测的更短传输来解释,贡献才成立。

这套"自我证伪"的设计,是把它从"水文"变成"可发表工作"的核心。

## 6. 边界:什么是贡献,什么不是

**算贡献:**
1. 信息化源的时空流先验 + 实测的"更短传输"。
2. 在(可能依赖的)耦合下、用于时空逆问题的算子无关引导。
3. 联合引导重叠窗口的长序列重建。
4. 能分离因果的速度-精度研究(打赢 S3GM-PF-ODE)。
5. 至少一项额外贡献:残差自适应引导 / 自适应 NFE 分配。

**不算贡献 / 明确非目标(第一版论文):**
- 跨无关动力系统的通用基础模型;
- 精确强制控制方程;
- 任意非线性 `H` 的后验精确性证明,或 `P ≠ 1` 的新理论;
- 跨系统零样本泛化的声称。

## 7. 路线图(里程碑概览)

每个里程碑是一道**门(gate)**,前一道不过不进下一道。

```
M0  仓库与可复现性(种子/配置/日志 + CPU 冒烟测试)
M1  KSE 数据管线(冻结划分、可逆归一化、窗口、测量算子)
M2  无条件流先验(线性 FM、先小批过拟合再全量训练)
M3  无引导先验评估(10/20/50/100 NFE,两点相关合理)
M4  单窗口稀疏引导(S3FM-Gauss):g_cov-G 让残差和 nRMSE 下降
─────────  以上都在“diffusion 等价的高斯源对照”上打通管线  ─────────
M4b 信息化源先验(S3FM-Info)★头号贡献:匹配 NFE 下打赢 M4,且实测传输更短
M5  并行重叠窗口:有 J_seq 时接缝更小
M6  自回归预测:误差增长曲线稳定、统计指标诚实
M7  速度-质量基准(含 S3GM-PF-ODE 归因基线):加速不只来自 SDE→ODE
M8  消融(源/耦合/引导 schedule/稀疏度…)
M9  更多系统(Kolmogorov 流 → ERA5 → 圆柱绕流)
```

## 8. 三条最重要的正确性检查(贯穿全程)

1. **端点:** 用 oracle 速度时,`X1_hat` 必须精确恢复干净目标(高斯源和信息化源都要测)。
2. **引导方向:** 一小步纯引导必须**降低**所定义的能量(一维玩具测试,数值验证符号)。
3. **时间语义:** 物理帧索引 `n` 和流积分时间 `s` **永远不能共用一个变量**。

## 9. 项目成功的定义(全部满足才算成功)

1. 流先验能生成统计上可信的无条件 KSE 动力学;
2. 引导样本匹配稀疏测量,且全场重建优于无引导;
3. **S3FM-Info 在匹配 NFE 下打赢 S3FM-Gauss,且有传输位移/曲率下降来解释;**
4. **NFE ≤ 50 时,S3FM 在相同 NFE 下打赢 S3GM-PF-ODE(优势不只是 SDE→ODE);**
5. 长序列无不可接受的接缝;
6. 自回归数值稳定、误差累积诚实报告;
7. 计入引导梯度后的真实墙钟时间,相对 S3GM 有意义地改善;
8. 精度-速度权衡在多个 NFE 上报告,而非只挑一个有利点;
9. 所有主张只限于实测过的系统与测量条件。
